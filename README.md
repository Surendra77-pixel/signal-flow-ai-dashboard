## вҡЎ What Is SignalFlow??

**SignalFlow** is an intelligent, real-time analytics dashboard that detects, analyzes, and diagnoses user drop-offs during complex signup flows. It goes beyond traditional funnel analytics by reading **behavioral signals** вҖ” rage clicks, hesitation patterns, tab switches вҖ” to pinpoint the *exact* reason users abandon your flow.

When a critical drop is detected, SignalFlow's built-in **AI Root-Cause Engine** maps the failure to a specific UX friction point and hands you an **actionable code patch** on the spot..

**Step by step:**

1. **Simulated Event Engine** вҖ” The backend simulates 8 different signup flows, generating telemetry that mimics real user behavior at scale.
2. **Behavioral Tracking** вҖ” Psychological and technical friction signals (rage clicks, hesitation, tab switches) are captured and timestamped.
3. **SDK Polling** вҖ” `mockApi.js` polls `/api/v1/metrics` endpoints rapidly to pull the latest drops and UX anomalies.
4. **React Dashboard** вҖ” Live data is visualized through interactive Recharts components showing funnel health in real time.
5. **AI Diagnosis** вҖ” On critical drop detection, the Insights Engine identifies the root cause and produces a direct code fix.

## рҹҡҖ Getting Started

### Prerequisites

- Node.js `v18+`
- npm `v9+`

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/Surendra77-pixel/signal-flow-ai-dashboard.git
cd signal-flow-ai-dashboard

# 2. Install all dependencies
npm install

# 3. Launch both servers concurrently
npm run dev
```

> рҹ–ҘпёҸ **Frontend** вҶ’ `http://localhost:3200`
> вҡҷпёҸ **API Server** вҶ’ `http://localhost:3001`

The `concurrently` package handles launching both the Vite dev server and the Node API in a single terminal session.

## рҹҸ—пёҸ Development Journey

**Phase 1 вҖ” Architecture & API Design**
Built a heavy Node/Express backend capable of producing streaming mock data indistinguishable from real-world telemetry вҖ” incorporating varied drop rates and complex multi-step funnel definitions.

**Phase 2 вҖ” Dashboard UI Construction**
